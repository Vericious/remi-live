# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities via public GitHub issues.
Open a private security advisory via GitHub's security tab instead.

## For contributors and AI agents

- Never commit API keys, tokens, bearer secrets, or credentials
- Never hardcode real IP addresses, internal hostnames, or personal data
- The feed URL and worker endpoints are public by design — do not commit auth tokens
- Use placeholder values in comments and examples
